src/data/questions.ts 파일의 모든 퀴즈 문제에 대해 정답의 정확성을 검증해 주세요.

## 작업 절차

### 1단계: 전체 문제 로드

src/data/questions.ts를 읽고 모든 문제를 파악합니다.

### 2단계: 각 문제별 정답 검증

모든 문제에 대해 다음을 확인합니다:

#### 2-1. 정답 일치 확인
- `answer` 인덱스가 가리키는 `options` 항목이 실제 정답과 일치하는가?
- `explanation`에서 설명하는 정답과 `options[answer]`가 일치하는가?

#### 2-2. 사실 정확성 확인
- 문제에서 주장하는 사실이 정확한가?
- 정답으로 지정된 보기가 실제로 맞는가?
- 오답 보기가 실제로는 정답이 될 수 있는 경우는 없는가?

#### 2-3. 해설 정확성 확인
- explanation의 내용이 사실에 부합하는가?
- 연도, 수치, 인명 등 구체적 정보가 정확한가?

### 3단계: 결과 출력

아래 형식으로 검증 결과를 출력합니다.

#### 문제가 있는 경우

```
## 정답 검증 결과

총 N개 문제 중 X개 문제에서 이슈 발견

### 이슈 목록

| ID | 카테고리 | 문제 요약 | 이슈 유형 | 상세 내용 | 수정 제안 |
|----|---------|----------|----------|----------|----------|
| | | | 정답 오류 / 사실 부정확 / 해설 오류 / 정답 논란 | | |
```

#### 문제가 없는 경우

```
## 정답 검증 결과

총 N개 문제 검증 완료 — 이슈 없음
```

### 4단계: 카테고리별 요약

```
## 카테고리별 요약

| 카테고리 | 문제 수 | 이슈 | 결과 |
|---------|--------|------|------|
| 한국사 | | | OK / 주의 |
| 과학 | | | OK / 주의 |
| 지리 | | | OK / 주의 |
| 일반상식 | | | OK / 주의 |
```

## 주의사항

- 시간에 따라 변할 수 있는 정보(순위, 통계 등)는 문제에 명시된 기준 시점 기준으로 판단합니다.
- 기준 시점이 명시되지 않은 경우, 이를 이슈로 보고합니다.
- 학술적 논란이 있는 내용은 주류 학설 기준으로 판단하되, 논란 여부를 비고에 기재합니다.
- 확실하지 않은 사실은 추측하지 말고 "확인 필요"로 표기합니다.
