다양한 플레이 시나리오를 시뮬레이션하여 점수 분포와 등급 결과를 예측해 주세요.

## 분석 대상 파일

- `src/utils/scoring.ts` — 점수 계산 함수 및 상수
- `src/components/ResultScreen.tsx` — 등급 판정 로직 (getGrade 함수)
- `src/data/questions.ts` — 문제 수, 카테고리 구성 확인

## 작업 절차

### 1단계: 채점 기준 확인

시뮬레이션에 필요한 채점 상수와 등급 기준을 확인합니다.

- 기본 점수, 시간 보너스, 최대 점수 등 scoring.ts 상수
- 등급 기준 (정답률 기준) ResultScreen.tsx getGrade 함수
- 총 문항 수, 카테고리당 문항 수

### 2단계: 전체 점수 시나리오 시뮬레이션

```
## 전체 점수 시나리오

| 시나리오 | 정답 수 | 평균 응답시간 | 총점 | 등급 | 비고 |
|---------|--------|------------|------|------|------|
| 완벽 (만점) | 40/40 | 즉시 (0초) | | S | 이론적 최고 |
| 우수 (빠름) | 36/40 | 5초 | | | |
| 우수 (보통) | 36/40 | 10초 | | | |
| 양호 (빠름) | 28/40 | 5초 | | | |
| 양호 (보통) | 28/40 | 10초 | | | |
| 보통 (빠름) | 20/40 | 5초 | | | |
| 보통 (느림) | 20/40 | 15초 | | | |
| 부진 | 12/40 | 10초 | | | |
| 최저 (전부 오답) | 0/40 | - | | D | 이론적 최저 |
```

점수 계산식: 정답 수 × (기본점수 + 남은시간 × 초당보너스)

### 3단계: 등급 경계값 시뮬레이션

각 등급의 경계에서 필요한 최소 조건을 산출합니다.

```
## 등급 경계값

| 등급 전환 | 필요 정답률 | 필요 정답 수 | 최소 점수 (느린 응답) | 최대 점수 (빠른 응답) |
|----------|-----------|------------|-------------------|-------------------|
| D → B | ≥50% | | | |
| B → A | ≥70% | | | |
| A → S | ≥90% | | | |

※ 등급은 정답률 기준이므로, 같은 등급 내에서도 시간 보너스에 따라 점수 차이 발생
```

### 4단계: 카테고리 편중 시나리오

특정 카테고리에 강하거나 약한 경우의 결과를 시뮬레이션합니다.

```
## 카테고리 편중 시나리오

| 시나리오 | 한국사 | 과학 | 지리 | 일반상식 | 총 정답 | 총점 | 등급 |
|---------|-------|------|------|--------|--------|------|------|
| 균등형 (7/10씩) | 7 | 7 | 7 | 7 | 28 | | |
| 한국사 특화 | 10 | 5 | 5 | 8 | 28 | | |
| 과학 취약 | 8 | 2 | 8 | 10 | 28 | | |
| 극단 편중 | 10 | 10 | 0 | 8 | 28 | | |
| 고른 우수 | 9 | 9 | 9 | 9 | 36 | | |
| 편중 우수 | 10 | 10 | 10 | 6 | 36 | | |

※ 응답 시간은 모두 10초로 통일하여 계산
```
