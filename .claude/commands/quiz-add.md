src/data/questions.ts 파일에 새로운 퀴즈 문제를 추가해 주세요.

## 입력 파라미터

- **카테고리**: $1 (한국사 | 과학 | 지리 | 일반상식)
- **난이도**: $2 (상 | 중 | 하)

## 난이도 기준

- **상**: 전문 지식이 필요하거나, 선택지 간 변별이 어려운 문제
- **중**: 일반 교양 수준에서 풀 수 있지만 약간의 사고가 필요한 문제
- **하**: 대부분의 사람이 쉽게 풀 수 있는 기본 상식 문제

## 작업 절차

### 1단계: 기존 문제 분석

src/data/questions.ts를 읽고 다음을 확인:

- 해당 카테고리의 기존 문제 목록 (중복 주제 방지)
- 현재 마지막 ID 번호 (새 문제의 ID = 마지막 ID + 1)
- 해당 카테고리의 기존 정답 분포 (answer 0~3 각각 몇 개인지)

### 2단계: 문제 작성

다음 형식으로 문제 1개를 작성:

```typescript
{
  id: number,
  category: '$1',
  question: '문제 내용',
  options: ['보기1', '보기2', '보기3', '보기4'],
  answer: number, // 0~3
  explanation: '해설 내용',
},
```

### 3단계: 교차 검증 가이드라인 적용

문제 작성 시 반드시 다음을 확인하고 검증 결과를 출력:

#### 3-1. 정답 유일성
- 정답이 하나뿐인가?
- 다른 보기가 정답이 될 수 있는 여지는 없는가?
- 애매한 경우 조건을 명시했는가?

#### 3-2. 최상급 표현 검증
- "가장", "최초", "최대", "최고", "최다", "유일" 등의 표현이 있는 경우:
  - 측정 기준이 명시되어 있는가?
  - 시점이 명시되어 있는가? (변할 수 있는 정보인 경우)
  - 지리적/분류적 범위가 한정되어 있는가?

#### 3-3. 시의성 확인
- 시간에 따라 변할 수 있는 정보는 시점(연도)을 명시했는가?
- 현재 시점에서 정답이 정확한가?

#### 3-4. 오답 보기 품질
- 오답 보기가 그럴듯한가? (너무 쉽게 배제되지 않는가)
- 오답 보기가 해당 분야와 관련성이 있는가?

### 4단계: 정답 분포 균형

해당 카테고리에서 가장 적게 사용된 정답 번호를 확인하고, 새 문제의 정답 위치를 해당 번호로 배치. 보기 순서를 조정하여 정답 분포 균형을 맞춤.

### 5단계: 코드에 삽입

- 해당 카테고리 섹션의 마지막 문제 뒤에 새 문제를 삽입
- 기존 코드 스타일(들여쓰기, 쉼표 등)을 정확히 따름

## 출력 형식

문제 추가 후 다음을 출력:

```

```
